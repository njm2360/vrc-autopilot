# PoseTelemetryHUD シェーダー仕様

VRChat のアバターに載せたシェーダー(PoseTelemetryHUD)が、視点のグローバル座標と向き
(6DoF)を画面左上に白黒ビットグリッドとして HUD 描画する。デコーダ(`app`)はこれを
スクリーンキャプチャで読み取り、座標フィードバック制御(OSC で移動・視点操作を注入)に使う。

デコーダ実装はこの仕様に準拠すること。定数は `app/perception/spec.py` のモジュール定数に
一元管理されている(`BLOCK` / `OFFSET_X` / `OFFSET_Y` をシェーダーの `_BlockPx` /
`_OffsetX` / `_OffsetY` と一致させる)。

## 描画条件

- VRChat のメイン画面(一人称視点)にのみ描画される。
  - `_VRChatCameraMode != 0` または `_VRChatMirrorMode != 0` では非描画。
  - つまり VRChat 内蔵スクリーンショットやハンドカメラ、ミラーには写らない。
- アンチエイリアスとポストプロセスを切ったワールドで運用する前提。
- HUD の表示はアバターの非同期(non-synced)パラメータ `HUD_Enable`(Bool, default=false)で制御。
  - OSC アドレス `/avatar/parameters/HUD_Enable` に true/false を送って ON/OFF できる。

## グリッドレイアウト

- 位置: 画面左上から `(OFFSET_X, OFFSET_Y) = (8, 8)` ピクセル。
- ブロック: 1ビット = `BLOCK x BLOCK = 4x4` ピクセルの正方形。白=1、黒=0。
- 構成: 32列 x 12行。1行 = 1ワード(32bit)。ビット順は **MSB が左端の列**。
- グリッド全体は `64 x 24` ピクセル。

## ワード定義(行番号順)

| 行   | 内容                                                                             |
| ---- | -------------------------------------------------------------------------------- |
| 0    | `MAGIC = 0x5AC3E7A1`(同期・検証用)                                               |
| 1    | `_VRChatTimeNetworkMs`(uint, ネットワーク同期時刻ms。フレーム識別用。ラップあり) |
| 2-4  | カメラ位置 x, y, z(float32 の IEEE754 ビットパターンをそのまま格納)              |
| 5-7  | カメラ前方ベクトル x, y, z(ワールド空間, 正規化済み, float32 ビットパターン)     |
| 8-10 | カメラ上方ベクトル x, y, z(同上)                                                 |
| 11   | 行0〜10の XOR チェックサム                                                       |

## 座標系

- Unity 準拠: Y-up、左手系。位置の単位はメートル。
- `yaw = atan2(fwd.x, fwd.z)` [+Z基準]、`pitch = asin(fwd.y)` [上向きが正]。
- デスクトップモードではロールは常に0だが、upベクトルも格納されている(将来のVR対応用)。

## デコード時の検証手順(必須)

1. 各ブロックは中心ピクセルをサンプリング(AAなし前提なので1点でよい)。
2. 白黒判定は RGB 合計のしきい値(中間値でよい。純白/純黒しか出ない)。
3. `word[0] == MAGIC` を確認。不一致なら読み取り失敗としてそのフレームを破棄。
4. `word[0]^...^word[10] == word[11]` を確認。不一致なら破棄。
5. `word[1]` が前回と同一なら同一フレームの二重読みとしてスキップ。
